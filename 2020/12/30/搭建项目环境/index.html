2020-12-30 # Java

Java搭建项目环境

1、file -> new Project -> 取名（webProject）

2、右击webProject -> Add Framework Support ->选中Web Appliacation ->OK （此时会多出一个web包）

3、点开WEB-INF -> 点开web.xml （此处可进行一些配置设置），此时不用配置

4、点击工具栏靠右上角的 Add Condiguration ->点击左上角的“+”号 -> 找到 Tomcat Server -> 选择 Local

右边 部分：

Name（原本为Tomcat +版本号，可改为Tomcat）

在Server里面：

Application server 那一行的右边，点击Configure ->点击Tomcat Home右边的文件图标，选择我们下载的Tomcat解压文件的位置 ->OK -> OK

取消选择After launch

在Deployment里面：

点击右边的“+”号 -> Artifact -> 下面的Application context里的路径，为了简便，可修改为只剩 / -> Apply ->OK

再次点击工具栏靠右上角的 Add Condiguration（此时已经变成了Tomcat，即我们刚刚所改我的Name）-> Edit Configuration -> 回到Server里面：

On ‘Update’ action 选择 Update classes and resources

On frame deactivation 选择 Update classes and resources

设置了之后刷新浏览器就能及时更新被修改的页面内容

JRE 选择 1.8（根据版本选择）

点击Apply

OK

5、此时运行web->WEB-INF->index.jsp，能看到页面已经能显示内容了

6、在web包下 new ->Directory ->取名为lib （存放servlet-api.jar)

7、找到下载的jar文件，解压，找到servlet-api.jar文件，复制

8、回到项目目录下，点击lib ->粘贴（刚刚复制的文件）-> OK

9、点开lib，可看到servlet-api.jar，单击右键->Add as Library ->OK

10、在web包下 new ->Directory ->取名为jsp

11、在jsp下new -> JSP/JSPX -> 取名（做登录界面，取名为login.jsp）-> OK

12、在显示的login.jsp下设计实现（登录）页面效果

13、若要进行样式的修改，在<head>标签下增加一个<style></style>标签，在其中更改样式

14、运行显示登录页面的内容，此时响应的是初始的index.jsp页面，此时我们在地址栏端口号8080后面加上登录页面的文件路径（/jsp/login.jsp），即可响应登录页面内容

此时修改之前写的service类

通过req.getParameter("method");获取前端传递的method值，通过此值判断执行哪个方法（此处设的method值为login，表明实现登录功能）

在前端的form表单提交处的action里的URL后加上?method=login

然后通过判断method的值，决定跳转到哪个功能模块下执行，此处为前面所写的toLogin方法（即登录功能）

【补】：直接写了方法后（会报错），鼠标点击报错内容，快捷键Alt+回车，选择created method，即可快速生成一个方法类，同时生成方法之后，要记得改成public的（根据需求决定是否更改）
